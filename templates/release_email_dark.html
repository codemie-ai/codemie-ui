EPAM AI/Run CodeMie Updates
Dear EPAM AI/Run CodeMie users,

We're excited to share the latest advancements in EPAM AI/Run CodeMie version 1.4.0! Our goal is to constantly innovate and improve your experience. Here's what's new:
⚡️ New Features and Enhancements
LangGraph Supervisor Agent Integration

Introduced the LangGraph Supervisor agent to manage workflows, enhancing efficiency and retiring the older autonomous workflow system in CodeMie. This improvement optimizes workflow management and resource allocation.
🪲 Fixes
  • Search Error Handling Improvements: Resolved an issue where searches with special characters (e.g., “C#”) triggered invalid filter errors in the Marketplace, significantly enhancing search reliability.

  • File-Type Detection for PPTX: Fixed MIME type assignment errors for PPTX files under the FILE_STORAGE_TYPE=filesystem setting, ensuring seamless file processing

  • Workflow and Operation Cancellation: Addressed issues where workflows or chat operations ran indefinitely post-cancellation, aligning the behavior with user expectations

  • ServiceNow Integration Fixes: Fixed an issue where unsupported arguments in API calls caused ServiceNow integration test failures, enabling smoother integration processes.
🎁 Additional Improvements

Made in hotfix Prod 1.3.1 include fixes addressing MIME type error messages during file uploads in chats, resolving iteration state failures with the Claude 3.7 model in workflows, and enhancing dynamic template processing with centralized argument handling for workflow nodes. These updates boost reliability, streamline workflows, and improve user interaction during development.
For further insights into this update, visit our  Release Notes.
